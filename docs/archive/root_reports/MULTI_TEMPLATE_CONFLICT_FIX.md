## 🎯 为什么会有多处冲突？

### 历史原因

1. **代码演进**：项目在不同阶段使用了不同的格式
   - 早期：16个节的复杂格式
   - 中期：7个节的简化格式（但第7个节是"质量检查清单"）
   - 现在：7个节的统一格式（第7个节是"衔接设计"）

2. **多人协作**：不同开发者可能修改了不同的文件
   - 有人修改了`prompt_definitions.py`
   - 有人修改了`blueprint.py`
   - 有人修改了`Novel_architecture.txt`
   - 但**没有同步修改**所有相关文件

3. **引用关系复杂**：
   - `blueprint.py`引用`prompt_definitions.py`
   - `chunked_chapter_blueprint_prompt`指向架构文件的Section 9
   - 多处定义，难以维护

### 设计问题

**单一职责原则违反**：
- 章节格式在多个文件中定义
- 修改一个地方，需要同步修改多个地方
- 容易遗漏，导致不一致

**更好的设计**：
```python
# 只在一处定义格式
CHAPTER_BLUEPRINT_FORMAT = """
7个节的标准格式...
"""

# 其他文件都引用这个定义
from chapter_format import CHAPTER_BLUEPRINT_FORMAT
```
