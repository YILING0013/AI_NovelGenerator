# 🚀 蓝图生成格式修复方案

## 执行时间
2026-01-11 01:15

## 目标
彻底解决蓝图生成中的格式混乱、重复问题，从源头修复。

---

## 根本原因回顾（简略）

### 1. Prompt 格式规范冲突
- 多个版本的格式规范共存（范例、模板、要求）
- 字段不一致（多余/缺失）
- Markdown 层级混乱

### 2. 范例文本被截断
- 第7节内容缺失
- 导致 LLM 误以为只需要6个节

### 3. 符号混用
- 方括号 vs 方头括号: `[]` vs `【】`
- 英文冒号 vs 中文冒号: `:` vs `：`
- 星号列表 vs 方括号列表: `*` vs `[]`

### 4. 重复和冗余的警告
- 同样的警告重复 8+ 次
- 噪音干扰，降低关键信息权重

### 5. 验证规则与 Prompt 不一致
- 期望格式与 Prompt 示例不匹配
- 导致 LLM 输出始终无法通过验证

---

```python
def validate_blueprint_v3(content, start_chapter, end_chapter):
    """验证蓝图格式 V3"""

    errors = []
    warnings = []
    suggestions = []

    # 1. 验证章节标题格式
    title_pattern = r'^第\d+章 - .+$'
    title_match = re.search(title_pattern, content, re.MULTILINE)
    if not title_match:
        errors.append("章节标题格式错误：应为'第X章 - 标题'")

    # 2. 验证所有7个节是否存在
    required_sections = [
        "## 1. 基础元信息",
        "## 2. 张力与冲突",
        "## 3. 匠心思维应用",
        "## 4. 伏笔与信息差",
        "## 5. 暧昧与修罗场",
        "## 6. 剧情精要",
        "## 7. 衔接设计",
    ]

    for section in required_sections:
        if section not in content:
            errors.append(f"缺失必需的节：{section}")

    # 3. 验证不重复
    lines = content.split('\n')
    chapter_numbers = []
    for line in lines:
        match = re.match(r'^第(\d+)章', line)
        if match:
            chapter_numbers.append(int(match.group(1)))

    # 检查是否有重复
    unique_numbers = set(chapter_numbers)
    for num in unique_numbers:
        count = chapter_numbers.count(num)
        if count > 2:
            errors.append(f"章节重复：第{num}章出现了{count}次")

    return {
        "is_valid": len(errors) == 0,
        "errors": errors,
        "warnings": warnings,
        "suggestions": suggestions
    }
```

**关键改进**:
1. ✅ 验证规则与 Prompt 格式要求完全一致
2. ✅ 提供清晰的错误信息
3. ✅ 避免模糊的匹配规则
