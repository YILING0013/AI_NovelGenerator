# 🎯 根本原因分析：章节混叠问题

## 用户问题

**现象**：第1章内容里出现了第2章的标题，导致格式混乱。

**用户质问**："为什么会有这个现象，不要总是靠质量检查，为什么不一上来就根除最根本的问题呢？"

## 🔍 根本原因定位

### 问题1：模板变量未替换 ❌

**位置**：`novel_generator/blueprint.py:431`

**错误代码**：
```python
{ENHANCED_BLUEPRINT_TEMPLATE}
```

**问题**：
- `ENHANCED_BLUEPRINT_TEMPLATE` 是从 `prompt_definitions.py` 导入的常量
- 包含模板变量：`{chapter_number}`
- **直接插入到prompt中**，没有替换变量
- LLM看到的是：`### **第{chapter_number}章 - [章节标题]**`
- 而不是：`### **第1章 - [章节标题]**`

**影响**：
- LLM困惑，不知道该填什么值
- 可能产生混乱的格式

### 问题2：模板示例不清晰 ❌

**位置**：`prompt_definitions.py:915-930`

**模板内容**：
```python
ENHANCED_BLUEPRINT_TEMPLATE = """
### **第{chapter_number}章 - [章节标题]**

## 1. 基础元信息
*   **章节序号**：第{chapter_number}章  ← 变量！
*   **章节标题**：[章节标题]
...
"""
```

**问题**：
- 使用了 `{chapter_number}` 变量，但这是Python模板语法
- LLM无法理解这是什么意思
- 没有说明"X"应该替换成什么数字

### 问题3：缺乏明确的批次要求 ❌

**缺失**：
- 没有明确告诉LLM"本次生成哪些章节"
- 没有强调"每章必须完整独立"
- 没有禁止"章节混叠"的明确规则

**影响**：
- LLM可能认为可以在章节A中预览章节B的内容
- 导致第1章内容中出现第2章的标题

## 🛠️ 根本修复方案

### 修复1：移除模板变量引用

**位置**：`novel_generator/blueprint.py:430-464`

**修复前**：
```python
1. **必须使用增强版模板**：
{ENHANCED_BLUEPRINT_TEMPLATE}  # ← 包含未替换的变量
```

**修复后**：
```python
1. **章节格式规范**：

每个章节必须遵循以下结构（按顺序）：

   ### **第X章 - [章节标题]**
   ## 1. 基础元信息
   *   **章节序号**：第X章
   *   **章节标题**：[章节标题]
   ...

   🚨 **格式禁忌**：
   - **严禁**在"基础元信息"中重复写"第X章 - 标题"
   - **严禁**在正文中引用具体章节号
```

**改进**：
- ❌ 不再使用包含变量的模板
- ✅ 直接在prompt中写出格式规范
- ✅ 使用 `X` 作为占位符，LLM可以理解
- ✅ 明确列出所有格式禁忌

### 修复2：添加明确的批次要求

**新增内容**：
```python
4. **批次要求**：
   - 本次生成第{start_chapter}章到第{end_chapter}章（共{end_chapter - start_chapter + 1}章）
   - **严格按顺序生成**，不得跳跃或重复
   - **每章必须完整独立**，不得出现"第1章内容中混入第2章开头"的情况
```

**改进**：
- ✅ 明确告诉LLM本次生成的范围
- ✅ 使用具体数字（1-5），而不是变量
- ✅ 强调每章必须完整独立
- ✅ 明确禁止章节混叠

### 修复3：强调章节完整性

**新增内容**：
```python
🚨 **格式禁忌**：
- **严禁**在"基础元信息"中重复写"第X章 - 标题"
- **严禁**在正文中引用具体章节号（如"第1章"、"第50章"）
- 只在章节开头写一次标题，后续用"本章"代替
- 引用其他章节时，用"后续章节"、"前文"代替
```

**改进**：
- ✅ 用"严禁"强化禁令
- ✅ 给出具体示例
- ✅ 提供替代方案

## 技术细节

### Python变量 vs LLM占位符

**错误的混用**：
```python
# Python代码
template = "第{chapter_number}章"  # Python变量
# 发送给LLM：第{chapter_number}章  # ❌ LLM不理解
```

**正确的做法**：
```python
# 直接使用LLM可理解的占位符
prompt = "每章格式：第X章 - 标题"  # ✅ LLM理解X是占位符
```

### Prompt层级结构

**新Prompt的3层结构**：

1. **格式规范**（第1层）
   - 明确章节结构
   - 列出所有字段
   - 给出具体示例

2. **格式禁忌**（第2层）
   - 用"严禁"强化禁令
   - 列出所有禁止行为
   - 提供替代方案

3. **批次要求**（第3层）
   - 明确本次生成范围
   - 强调完整独立性
   - 禁止章节混叠
