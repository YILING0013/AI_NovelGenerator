# 🎯 从源头控制：Prompt模板改进总结

## 设计理念

**漂亮的代码**不是事后修复，而是从一开始就控制好生成质量。

## 问题诊断

### 原始问题
LLM在生成时会产生重复章节：
```
第7行: 第1章 - 标题              ← 章节标题
第10行: 章节：第1章 - 标题       ← 重复！
```

### 根本原因
**Prompt模板有问题**！
```python
# ❌ 旧模板（会导致重复）
*   **章节**：第[X]章 - [章节标题]

# LLM理解为：需要在"基础元信息"中重复写章节标题
```

### 解决方案
**改进Prompt模板**，从源头控制LLM生成正确格式。

## 改进措施

### 改进1：修复ENHANCED_BLUEPRINT_TEMPLATE

**位置**：`prompt_definitions.py:915-930`

**修复前**：
```python
ENHANCED_BLUEPRINT_TEMPLATE = """
### **第{chapter_number}章 - [章节标题]**

## 1. 基础元信息
*   **章节**：第[X]章 - [章节标题]  ← 重复！
*   **定位**：第[X]卷 [卷名] - 子幕[X] [子幕名]
...

```

**修复后**：
```python
ENHANCED_BLUEPRINT_TEMPLATE = """
### **第{chapter_number}章 - [章节标题]**

## 1. 基础元信息
*   **章节序号**：第{chapter_number}章  ← 分开写
*   **章节标题**：[章节标题]           ← 分开写
*   **定位**：第[X]卷 [卷名] - 子幕[X] [子幕名]
...

🚨 **格式禁忌**：
- **严禁**在基础元信息中重复写"第X章 - 标题"
- **严禁**在正文中引用章节号（如"第1章"）
- 只在章节开头写一次标题，后续用"本章"代替
"""
```

### 改进2：添加明确的格式规范

**位置**：`novel_generator/blueprint.py:374-381`

**新增内容**：
```python
4. **🚨 格式规范（严格执行）**：
   - **章节标题格式**：每章开头只写一次"第X章 - 标题"
   - **基础元信息**：分开写"章节序号"和"章节标题"，不要合并
   - **禁止重复标题**：不要在内容中重复写"第X章 - 标题"
   - **引用替代**：正文中需要引用其他章节时，用"后续章节"、"前文"代替，不要写具体章节号
   - **示例**：
     ✅ 正确："本章将在后续章节揭示秘密"
     ❌ 错误："本章将在第50章揭示秘密"
```

## Prompt设计原则

### 原则1：明确性 > 模糊性
```python
# ❌ 模糊：容易误解
*   **章节**：第[X]章 - [章节标题]

# ✅ 明确：清晰分离
*   **章节序号**：第{chapter_number}章
*   **章节标题**：[章节标题]
```

### 原则2：示例 > 规则
```python
# 提供正确和错误的示例
✅ 正确："本章将在后续章节揭示秘密"
❌ 错误："本章将在第50章揭示秘密"
```

### 原则3：禁止 > 允许
```python
# 明确列出禁止事项
🚨 **格式禁忌**：
- **严禁**在基础元信息中重复写"第X章 - 标题"
- **严禁**在正文中引用章节号
```

### 原则4：一致性 > 多样性
- 模板、Few-Shot示例、格式规范三处保持一致
- 所有示例都遵守同一套规则
- 避免LLM混淆

## 验证逻辑配合

虽然Prompt已经改进，但验证逻辑仍然保留作为**最后一道防线**：

```python
# 智能验证（宽松但有效）
- 只检测行首的章节标题
- 允许每个章节最多出现2次（正常+1次引用）
- 只拒绝过度重复（>2次）
```

**配合方式**：
1. **Prompt控制**：99%的情况下，LLM遵守规则，一次生成成功 ✅
2. **验证防守**：1%的边缘情况，验证拦截并要求重试 ⚠️
3. **去重保险**：即使有遗漏，批次后去重清理 🧹
