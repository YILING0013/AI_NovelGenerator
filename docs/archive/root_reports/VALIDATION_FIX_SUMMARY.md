# 🔧 验证逻辑修复总结

## 问题描述

新的验证逻辑太严格，把所有"第X章"的出现都算作重复，导致LLM生成的内容因为合理的章节引用（如伏笔、铺垫）而验证失败，经过5次重试后仍然失败。

## 错误信息

```
Exception: 第1批次生成失败: 批次生成失败：第1章到第5章，经过5次尝试仍未成功
```

## 根本原因

**旧验证逻辑**（过于严格）：
```python
# 匹配所有"第X章"的出现
chapter_pattern_all = r"第(\d+)章"
all_chapter_mentions = re.findall(chapter_pattern_all, content)
# 结果：把内容引用也算作重复
# 第1章: 3次（1次标题 + 2次引用）→ 失败 ❌
```

**问题**：
- LLM在内容中合理引用章节号（如"第50章揭示"、"第6章送药"）
- 旧逻辑把这些都算作重复，导致验证失败
- 实际上这些引用是正常的，不应该被拒绝

## 修复方案

### 新验证逻辑（智能区分）

```python
# 只匹配行首的章节标题
chapter_title_pattern = r"(?:^|\n)第\s*(\d+)\s*章(?:\s*[-–—]\s*[^\n]*|\n)"
chapter_titles = re.findall(chapter_title_pattern, content, re.MULTILINE)
generated_numbers = [int(ch) for ch in chapter_titles if ch.isdigit()]

# 智能重复检测
# - 1次：正常 ✅
# - 2次：可能是内容引用，可接受 ✅
# - 3次及以上：过度重复，拒绝 ❌
for num in unique_numbers:
    count = generated_numbers.count(num)
    if count > 2:  # 只拒绝超过2次的
        duplicate_numbers.append(num)
```

### 逻辑改进

| 检测对象 | 旧逻辑 | 新逻辑 | 效果 |
|---------|--------|--------|------|
| 检测范围 | 所有"第X章" | 只检测行首的章节标题 | 忽略内容引用 |
| 重复阈值 | >1次就失败 | >2次才失败 | 允许1次合理引用 |
| 验证结果 | 过于严格 ❌ | 宽松合理 ✅ | 正常通过 |

## 双重保险机制

### 层级1：智能验证（生成时）
- 只检测行首的章节标题
- 允许合理的章节引用
- 拒绝明显的过度重复（>2次）

### 层级2：批次去重（保存后）
- 每个批次保存后立即执行`_format_cleanup()`
- 清理所有重复（包括内容引用）
- 确保最终文件无重复

**优势**：
- 验证不会因为合理引用而失败
- 最终文件仍然通过去重保证无重复
- 用户体验更好（不会频繁重试）

## 正则表达式详解

### 行首章节标题匹配
```regex
(?:^|\n)              # 匹配行首或换行后
第\s*(\d+)\s*章       # 匹配"第X章"，允许空格
(?:                   # 非捕获组
  \s*[-–—]\s*[^\n]*   # 空格+破折号+标题（可选）
  |                   # 或
  \n                  # 直接换行
)
```

**匹配示例**：
- ✅ `第1章 - 致命死局与金缮初现`
- ✅ `第2章`
- ✅ `第 3 章 - 标题`（允许空格）
- ❌ `章节：第1章 - 标题`（不在行首）
- ❌ `[第50章揭示]`（在内容中）
