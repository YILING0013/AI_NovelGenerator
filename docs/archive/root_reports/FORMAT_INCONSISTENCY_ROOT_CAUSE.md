# 🎯 格式不一致问题的根本原因分析

## 用户反馈问题

**现象**：
1. 第1章内容里包含了第2-4章的内容
2. 从第2章开始格式就跟模板不一样了
3. 后面的章节开始偷工减料，省略了后面的节

**用户原话**："蓝图文件第一章还是包含1-4章，从第2章开始跟格式模版就不一样了，开始偷工减料。"

## 🎯 根本原因

### 原因1：Few-shot示例与Prompt要求不一致！

**Prompt要求的第7个节**：
```
## 7. 衔接设计
```

**Few-shot示例的第7个节**（prompt_definitions.py:1050）：
```
## 7. 质量检查清单  ← 不一致！
```

**影响**：LLM混淆了，不知道该用哪个名称。

### 原因2："已有章节概览"展示了错误的格式！

**Prompt结构**：
```
### 已有章节概览
[第239-240章的实际内容，使用错误的格式]

⚠️ **重要警告**：以上"已有章节概览"仅用于了解剧情连贯性，**严禁模仿其格式**！
你必须严格按照下方的标准格式（7个节）生成，不能省略任何节。
```

**问题**：虽然有警告，但是：
1. 警告是在之前版本中**没有**的（刚添加）
2. LLM看到具体的实例后，会**优先模仿实例**
3. 警告文字不如实例有说服力

**结果**：LLM模仿了"已有章节概览"的格式（只有3个节），而不是遵循标准格式（7个节）。

### 原因3：Few-shot示例不完整

**示例只展示了节标题**，没有展示每个节的具体内容格式：

```
## 2. 张力与冲突
## 3. 匠心思维应用
## 4. 伏笔与信息差
...
```

**LLM不知道每个节该怎么写**，所以它：
- 看到了"已有章节概览"中的"2. 剧情细纲与节点"
- 就模仿了这个格式

### 原因4：Prompt格式规范不够详细

**之前的Prompt**：
```
## 2. 张力与冲突
## 3. 匠心思维应用
...
```

只列出了节标题，没有说明每个节应该包含什么内容。

## 技术细节

### LLM的学习模式

**实例优先原则**：
- LLM会优先学习具体的实例（Example）
- 其次才是抽象的规则（Rule）
- 如果实例和规则冲突，LLM倾向于跟随实例

**本案例中的应用**：
- 实例："已有章节概览"中的3个节格式
- 规则：Prompt要求的7个节格式
- 结果：LLM跟随了实例，生成3个节

**解决方案**：
1. 统一实例和规则（修复Few-shot示例）
2. 明确警告不要跟随错误实例（添加警告）
3. 提供更详细的规则（展示每个节的内容格式）

### Prompt工程最佳实践

**避免实例与规则冲突**：
- ✅ Few-shot示例应该与要求完全一致
- ✅ 如果提供的实例格式不同，必须明确警告
- ✅ 最好不提供格式不同的实例

**提供完整的格式说明**：
- ✅ 不仅列出节标题，还要说明内容格式
- ✅ 给出具体的内容项，让LLM知道该怎么写
- ✅ 提供正反例对比

## 总结

**根本原因**：
1. Few-shot示例与Prompt要求不一致（第7个节名称不同）
2. "已有章节概览"展示了错误的格式（只有3个节）
3. LLM优先模仿实例而不是遵循规则
4. Prompt格式规范不够详细（只列出标题，没有内容格式）

**解决方案**：
1. ✅ 统一Few-shot示例与Prompt要求
2. ✅ 添加"严禁模仿已有章节格式"的警告
3. ✅ 详细展示每个节的内容格式
4. ✅ 强化完整性铁律（"严禁只写3个节"）

**设计原则**：
> 实例（Example）与规则（Rule）必须一致，如果无法一致，必须明确警告。
