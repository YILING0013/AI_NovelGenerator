# 蓝图生成格式混乱、重复根本原因分析报告

## 执行时间
2026-01-11 01:00

## 分析方法
- 检查 LLM 对话日志（`llm_conversation_logs/*.md`）
- 分析 Prompt 构建（`prompt_definitions.py`）
- 对比 Prompt 要求与 LLM 实际输出
- 找出格式不一致和重复的根本原因

---

## 根本原因分析

### 🚨 核心问题 1: **Prompt 格式规范冲突**

#### 问题描述
Prompt 中存在**多个版本**的格式规范，导致 LLM 混淆：

**版本 A - 范例中的格式**（`BLUEPRINT_FEW_SHOT_EXAMPLE`）:
```
### **第1章 - 绝境处刑与金缮初现**

## 1. 基础元信息
*   **章节序号**：第1章
*   **章节标题**：绝境处刑与金缮初现
*   **定位**：第1卷 凡胎重铸 - 子幕1 乱葬岗觉醒
*   **核心功能**：...
*   **字数目标**：4500字
*   **出场角色**：...
*   **张力评级**：SS级（生死危机）  ← 多余字段！
*   **目标字数**：4500字           ← 重复字段！
```

**版本 B - 模板中的格式**（`ENHANCED_BLUEPRINT_TEMPLATE`）:
```
### **第{chapter_number}章 - [章节标题]**

## 1. 基础元信息
*   **章节序号**：第{chapter_number}章
*   **章节标题**：[章节标题]
*   **定位**：第[X]卷 [卷名] - 子幕[X] [子幕名]
*   **核心功能**：[一句话概括本章作用]
*   **字数目标**：[3000-5000] 字
*   **出场角色**：[列出本章角色]
```

**版本 C - 严格要求中的格式**（`strict_requirements`）:
```
每个章节必须遵循以下结构：
### **第X章 - [章节标题]**

## 1. 基础元信息
*   **章节序号**：第X章
*   **章节标题**：[章节标题]
*   **定位**：...
*   **核心功能**：...
*   **字数目标**：[3000-5000] 字
*   **出场角色**：...
```

#### 冲突点
1. **章节标题格式不一致**:
   - 范例使用: `### **第1章 - 绝境处刑与金缮初现**`
   - 模板使用: `### **第{chapter_number}章 - [章节标题]]**`
   - **差异**: 范例中标题包含完整信息，模板中是占位符

2. **多余字段**:
   - 范例中有 `张力评级` 和 `目标字数` 两个字段
   - 模板中只有 `字数目标` 一个字段
   - **LLM 会困惑**: 应该包含哪些字段？

3. **Markdown 层级混淆**:
   - 范例: `### **第X章 - 标题**` (三级标题 + 粗体)
   - 节标题: `## 1. 基础元信息` (二级标题 + 数字)
   - **LLM 难以理解**: 为什么标题用三级，节标题用二级？

---

### 🚨 核心问题 2: **范例文本被截断**

#### 问题描述
在多个日志中，范例文本在第 7 节处被截断：

**第21章日志（第43行）**:
```
## 7....             ← 被截断！
```

**影响**:
- LLM 看不到完整的第7节"衔接设计"
- 可能误以为只需要6个节
- 导致生成的蓝图缺少第7节

**原因分析**:
- `BLUEPRINT_FEW_SHOT_EXAMPLE` 可能太长（超过3000字符）
- 日志系统为了节省空间，截断了 Prompt
- 或者 Prompt 构建时只包含了部分范例

---

### 🚨 核心问题 3: **重复和冗余的警告信息**

#### 问题描述
Prompt 中有**大量重复**的警告和强调：

**重复次数统计**:
- "绝对禁止省略" - 出现 8+ 次
- "零容忍政策" - 出现 4+ 次
- "必须包含全部7个节" - 出现 6+ 次
- "严禁使用...格式" - 出现 5+ 次

**影响**:
- LLM 被"噪音"干扰，难以理解核心要求
- 过多的警告可能导致 LLM 反而忽略所有警告
- 降低了关键信息的"权重"

**示例**:
```python
# Prompt 中的重复
"1. **禁止任何形式的省略 - 零容忍政策**：
   - 绝对禁止使用"..."、"…"、"省略"、"跳过"等任何省略表述
   - 每一章都必须有完整的详细内容
   - **严禁生成多余章节**：只生成第1章到第1章，不要自动续写后续章节！"

# 后面又重复
"3. **格式严格一致性（Formatting Consistency）**:
    - 第1章到第1章，**每一章**都必须严格使用上述模板格式。
    - **每一章都必须包含全部7个节**：基础元信息、张力与冲突、匠心思维应用、伏笔与信息差、暧昧与修罗场、剧情精要、质量检查清单
    - **不得省略任何一节**，即使某节内容为"本章不涉及"也必须保留该节标题"

# 再后面又重复
"🚨【绝对强制性要求 - 违者视为任务失败】🚨
1. **禁止任何形式的省略 - 零容忍政策**：
   - 绝对禁止使用"..."、"…"、"省略"、"跳过"等任何省略表述"
   - 每一章都必须有完整的详细内容
```

### 🚨 核心问题 4: **符号混用**

#### 问题描述
Prompt 中使用了多种不同的符号表示相同的事物，导致 LLM 混淆：

**方括号 vs 方头括号**:
```
范例中: 【基础元信息】
要求中: ## 1. 基础元信息
```

**冒号 vs 中文冒号**:
```
字段名**: 值          (英文冒号)
字段名`：`值          (中文冒号)
```

**列表符号混用**:
```
*   **项目名**：值    (星号列表)
[伏笔内容] -> [预计在第X章揭示]  (方括号列表)
```

**影响**:
- LLM 可能随意选择一种符号
- 导致生成的内容格式不统一
- 验证正则表达式匹配失败

---

### 🚨 核心问题 5: **验证规则与 Prompt 要求不一致**

#### 问题描述
`strict_validation` 中的验证规则与 Prompt 中的格式要求不一致：

**Prompt 要求**:
```
### **第{chapter_number}章 - [章节标题]**

## 1. 基础元信息
*   **章节序号**：第{chapter_number}章
*   **章节标题**：[章节标题]
...
```

**LLM 实际输出**（从日志中）:
```
第1章 - 碎瓷     ← 没有 ### 和粗体

1. 基础元信息        ← 使用了 1. 而不是 ## 1.
章节序号：第1章      ← 没有粗体标记
...
```

**验证失败的原因**:
- 验证正则: `r'^## \d+\. .+$'` (期望 `## 1.`)
- LLM 输出: `1. 基础元信息` (没有 `##`)
- 匹配失败，导致重复生成

## 为什么会重复生成？

### 循环流程

```
1. Prompt 发送到 LLM (包含矛盾的格式规范)
   ↓
2. LLM 根据理解生成内容 (格式可能错误)
   ↓
3. 验证失败 (格式不匹配)
   ↓
4. 将错误信息加入 Prompt
   ↓
5. 重新生成 (LLM 可能尝试另一种格式)
   ↓
6. 再次验证失败 (仍然不符合规范)
   ↓
7. 重复 2-6 步，直到重试次数耗尽
```

### 重复的具体表现

从日志中看到的重复模式:

**1. 章节标题重复**:
```
第1章 - 碎瓷
第1章 - 绝境处刑与金缮初现  ← 重复！
...
```

**2. 节内容重复**:
```
## 1. 基础元信息
章节序号：第1章
章节标题：碎瓷
...
## 1. 基础元信息   ← 重复！
章节序号：第1章
章节标题：碎瓷
...
```

**3. 格式混用导致的验证失败**:
- 第一次尝试: 使用 `### **第X章 - 标题**`
- 第二次尝试: 使用 `第X章 - 标题`
- 第三次尝试: 使用 `### 第X章 标题`
- ...

## 总结

### 根本原因（按优先级）

1. **【最严重】Prompt 格式规范冲突**
   - 多个版本共存（范例、模板、要求）
   - 字段不一致（多余/缺失）
   - Markdown 层级混乱

2. **【严重】范例文本截断**
   - 第7节内容缺失
   - 导致 LLM 不知需要7个节

3. **【严重】符号混用**
   - 方括号 vs 方头括号
   - 英文冒号 vs 中文冒号
   - 星号列表 vs 方括号列表

4. **【中等】重复和冗余的警告**
   - 同样的警告重复 8+ 次
   - 噪音干扰，降低关键信息权重

5. **【中等】验证规则与 Prompt 不一致**
   - 期望格式与 Prompt 示例不匹配
   - 导致 LLM 输出始终无法通过验证

## 下一步行动

### 彻底修复方案（将实施）

1. **统一格式规范**
   - 删除所有冲突的格式版本
   - 只保留一个清晰的格式定义
   - 消除所有多余字段

2. **修复范例文本**
   - 确保范例完整，不被截断
   - 范例格式与要求完全一致
   - 只提供一个简短但完整的范例

3. **消除符号混用**
   - 统一使用中文冒号 `：`
   - 统一使用方头括号 `【】` 或方括号 `[]`
   - 统一使用星号列表 `*`

4. **精简 Prompt**
   - 删除所有重复的警告
   - 保留一次清晰的核心要求
   - 提高关键信息的权重

5. **统一验证规则**
   - 验证规则与 Prompt 格式要求完全一致
   - 避免模糊的匹配规则
   - 提供清晰的错误信息
