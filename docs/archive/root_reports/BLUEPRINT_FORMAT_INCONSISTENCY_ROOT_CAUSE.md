# 蓝图格式不一致的根本原因分析

## 📋 问题概述

**用户反馈**："每一章的项目都不一样，有重复的，还有混乱的。"

**实际表现**：
- 第1章：使用`2. 张力与冲突`、`3. 匠心思维应用`（部分符合模板）
- 第2章：使用`### 【基础元信息】`、`### 【张力架构】`（完全不同格式）
- 第3章：又混用了另一种格式

**严重程度**：🔴 严重 - 导致蓝图不可用，后续章节生成失败

---

## 🔍 问题根源分析

### 1. Prompt模板定义

**位置**: `prompt_definitions.py:915-990`

**标准格式**：
```markdown
### **第{chapter_number}章 - [章节标题]**

## 1. 基础元信息
*   **章节序号**：第{chapter_number}章
*   **章节标题**：[章节标题]
...

## 2. 张力与冲突
*   **冲突类型**：[生存/利益/理念/情感]
...

## 3. 匠心思维应用
...
```

**关键特征**：
- 章节标题：`### **第X章 - 标题**`（三级标题+粗体）
- 节标题：`## 1. 基础元信息`（二级标题+数字编号）
- 项目：使用`*   **项目名**：内容`格式

### 2. Few-Shot示例

**位置**: `prompt_definitions.py:992-1057`

**示例格式**：与模板一致，使用`## 1. 基础元信息`、`## 2. 张力与冲突`

### 3. LLM实际生成结果

#### 第1章（部分符合）
```markdown
第1章

定位：第1卷 凡胎浊骨 - 子幕1 乱葬岗惊变
```

- ❌ 项目名称完全不同：`章节核心`、`张力评级`、`目标字数`（不是模板中的`章节序号`、`章节标题`、`定位`）
- ❌ 新增了模板中没有的节：`【情感轨迹】`、`【核心结构】`、`【系统机制】`

#### 第3章（又一种格式）

- ❌ 使用了`-   **项目名**：内容`（无序列表）
- ❌ 项目名称又不一致

## 🎯 根本原因总结

### 原因1：Prompt约束力不足

**问题**：
- 模板定义很清晰，但LLM没有严格遵守
- Few-Shot示例只有1个（第1章），约束力不够
- 缺少"必须严格按照此格式"的强制性指令

**证据**：
- 第1章有部分符合（可能是因为Few-Shot示例是第1章）
- 第2章完全偏离（可能是LLM"忘记"了格式）
- 第3章又是一种变体（格式进一步退化）

### 原因2：LLM生成长内容时的格式漂移

**问题**：
- LLM在生成长文本时，往往会"忘记"早期的格式要求
- 随着内容增长，格式逐渐"漂移"到LLM更熟悉的模式
- 第2章、第3章的格式（`### 【xxx】`）可能是LLM更"自然"的输出方式

**分析**：
- `### 【基础元信息】`是三级标题+方括号，这种格式在LLM的训练数据中可能更常见
- LLM可能"认为"这种格式更好，于是"主动改进"了

### 原因3：缺少格式验证机制

**问题**：
- 代码中的验证逻辑（`_validate_generated_chapters`）只检查章节标记
- **没有验证章节内部的格式是否一致**
- **没有验证每个章节是否使用了相同的节结构**

**证据**：
- `blueprint.py:223-271`的验证只检测：
  - 章节标题是否存在
  - 章节编号是否连续
  - 章节内容长度是否足够
- **不检测**：
  - 节标题格式是否一致（`## 1. 基础元信息` vs `### 【基础元信息】`）
  - 项目名称是否一致（`章节序号` vs `章节核心`）

### 原因4：可能的多次LLM调用

**推测**：
- 蓝图可能是分批次生成的（每批50章）
- 每次调用LLM时，Prompt的"记忆"可能丢失
- 导致不同批次的章节格式不一致

**需要验证**：
- 检查`StrictChapterGenerator`的生成逻辑
- 确认是否为多次LLM调用
- 确认每次调用是否都传递了完整的模板

### 方案2：增强格式验证逻辑

**新增验证函数**：

```python
def _validate_chapter_format_consistency(self, content: str) -> tuple:
    """
    验证所有章节的格式是否一致

    Returns:
        (is_valid: bool, errors: list)
    """
    lines = content.split('\n')
    chapters = self._extract_chapters(lines)

    if len(chapters) < 2:
        return True, []  # 只有一章，无法比较

    # 提取第一章的格式作为标准
    first_chapter = chapters[0]
    standard_format = self._extract_chapter_format(first_chapter)

    errors = []

    # 检查其他章节是否遵循相同格式
    for i, chapter in enumerate(chapters[1:], 2):
        chapter_format = self._extract_chapter_format(chapter)

        # 比较节标题格式
        if chapter_format['section_headers'] != standard_format['section_headers']:
            errors.append(
                f"🚨 第{i}章的节标题格式与第1章不一致！"
                f"第1章: {standard_format['section_headers']}, "
                f"第{i}章: {chapter_format['section_headers']}"
            )

        # 比较项目名称
        if chapter_format['item_names'] != standard_format['item_names']:
            errors.append(
                f"🚨 第{i}章的项目名称与第1章不一致！"
                f"第1章: {standard_format['item_names']}, "
                f"第{i}章: {chapter_format['item_names']}"
            )

    return len(errors) == 0, errors

def _extract_chapter_format(self, chapter_lines: list) -> dict:
    """提取章节的格式信息"""
    format_info = {
        'section_headers': [],  # 节标题列表
        'item_names': [],      # 项目名称列表
    }

    for line in chapter_lines:
        # 提取节标题（如 "## 1. 基础元信息"）
        if re.match(r'^##\s+\d+\.\s+', line):
            format_info['section_headers'].append(line.strip())

        # 提取项目名称（如 "**章节序号**"）
        match = re.match(r'^[\*\-]\s*\*\*([^*]+)\*\*[:：]', line)
        if match:
            format_info['item_names'].append(match.group(1))

    return format_info
```
